# CLAUDE.md — Agent Protocol

Personal portfolio site for **Kelvin Benzali** (Senior Software Engineer, Tokopedia / ByteDance).
Single-person personal brand, one product surface: the site itself. Live at `www.kelvinbenzali.com`.

## Stack

- **Framework:** Next.js 14.2 (App Router, React Server Components)
- **Language:** TypeScript 5.5 (`strict: true`)
- **UI:** React 18.3, Tailwind CSS 3.4, Plus Jakarta Sans (`next/font/google`)
- **Data / auth:** Supabase (`@supabase/ssr`) — analytics events + email/OAuth auth
- **Icons:** `lucide-react` (UI glyphs), `react-icons/si` + `/fa` + `/bi` (brand marks)
- **Animation:** GSAP (hero typewriter only)
- **Hosting / telemetry:** Vercel, `@vercel/analytics`, `@vercel/speed-insights`

> The frontend is **Next.js, not plain React** — server components are the default; reach for `'use client'` only when a component needs browser APIs, state, or effects.

## Layout

```
app/            App Router routes + layout + globals.css
  page.tsx        Home (hero, about, work, contact, footer) — async RSC
  project/        Projects timeline page — async RSC
  login/ private/ Supabase-auth utility pages (bare, unbranded — intentionally so)
  auth/ api/      OAuth callback, signout, session-cookie route handlers
components/      Section + primitive components (About, Contact, Header, WorkCard, ...)
utils/lib/       Analytics + request helpers (event, req, country, user-agent, icon, ...)
utils/supabase/  SSR/client/route Supabase factories
middleware.ts    Issues sticky `sid` session cookie on every request
public/          Images, favicons, project media
```

## `./design-system` is NOT part of the codebase

It is a **design handover** generated by Claude Design — tokens, guideline cards, and a static UI-kit recreation. It is **reference material, not shipped code**: never import from it, never build against it. Treat it as the canonical intent for color/type/spacing; the actual Tailwind classes in `components/` are the implementation. See DESIGN.md for where the two diverge.

## Commands

```bash
npm run dev               # local dev server → http://localhost:3000
npm run build             # production build
npm start                 # serve production build
npm run lint              # next lint (eslint-config-next)
npm run generate:favicons # regenerate favicon assets
```

Requires `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` in the environment. Analytics writes are skipped on `localhost` / non-production, so the About dashboard shows empty data locally — this is expected.

## Working conventions

- **Server-first:** fetch data in async server components (see `app/page.tsx` → `AboutSection`). Keep client components small and leaf-level.
- **Style with Tailwind utilities**, not custom CSS. The only bespoke CSS lives in `app/globals.css` (`.container-max`, typing-dot keyframes).
- **Accent is emerald** (`emerald-400/500`), not the `brand` sky-blue in `tailwind.config.js` (declared but unused). See DESIGN.md.
- **Never commit secrets.** Supabase keys come from env, never hard-coded.
- Match the surrounding file's idiom, spacing, and comment density when editing.
